# 🎬 AI-Powered Science Animation System

An end-to-end multi-agent AI pipeline that transforms science articles into
narrated educational animations — automatically.

**Input**: Science article (text / PDF / URL)  
**Output**: Educational MP4 video with synchronized narration and Manim animations

---

## Architecture

```
User Input (article / PDF / URL)
         │
         ▼
┌─────────────────────────────────────────┐
│         Supervisor Agent                │
│     (LangGraph State Machine)           │
└──┬──────────┬──────────┬───────────────┘
   │          │          │
   ▼          ▼          ▼
Scriptwriter  Voiceover  Animator Agent
  Agent        Agent    (Manim + Docker)
   │            │           │
   └─────┬──────┘           │
         ▼                  ▼
    Storyboard JSON    Video Clips
         │                  │
         └───── FFMPEG ──────┘
                   │
                   ▼
           Draft Video
                   │
                   ▼
      HITL Review (Streamlit UI)
                   │
          ┌────────┴────────┐
          │                 │
       Revise           Approve
          │                 │
     [loop back]    Social Media Agent
                    (YouTube / Instagram)
```

## Quick Start

### 1. Prerequisites

- Python 3.11+
- Docker Desktop (for Manim rendering)
- FFMPEG installed on host
- API keys: OpenAI (required), ElevenLabs (optional)

### 2. Install

```bash
# Clone the repository
git clone <repo-url>
cd science-animation-system

# Install dependencies
make install

# Build the Manim Docker rendering image
make build-manim-image
```

### 3. Configure API Keys

```bash
cp .env.example .env
# Edit .env and fill in your API keys
```

Required keys:
- `OPENAI_API_KEY` — for GPT-4o and TTS
- `TTS_PROVIDER` — `openai` or `elevenlabs`

Optional keys (for social media publishing):
- `YOUTUBE_CLIENT_ID` / `YOUTUBE_CLIENT_SECRET`
- `INSTAGRAM_ACCESS_TOKEN` / `INSTAGRAM_ACCOUNT_ID`
- `ELEVENLABS_API_KEY` / `ELEVENLABS_VOICE_ID`

### 4. Build the RAG Knowledge Base

The RAG system grounds Manim code generation in official documentation.
LLM-generated examples are validated by actually rendering them before indexing:

```bash
# Step 1: Validate LLM-generated examples (runs each through Docker)
make validate-examples

# Step 2: Build the ChromaDB index (skips unvalidated examples)
make build-rag

# Or do both in one step:
make build-rag-full
```

> **Why validation matters**: The RAG examples were generated by an LLM and
> could contain hallucinated API calls. The validation step renders each
> example through the actual Manim runtime — only examples that successfully
> render are indexed. This prevents the RAG system from amplifying hallucinations.

### 5. Run

```bash
# Start the Streamlit HITL interface
make run
# → Open http://localhost:8501
```

Or run headlessly (CLI):
```bash
make run-headless TEXT="Quantum entanglement is a phenomenon..."
make run-headless PDF=lecture.pdf
make run-headless URL=https://arxiv.org/abs/...
```

### Key Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Orchestration | LangGraph | Explicit state machine, auditable flow |
| Rendering isolation | Docker sandbox | Prevents LLM-generated code from crashing host |
| Manim error handling | 5-retry self-correction loop | Covers >90% of hallucination patterns |
| TTS timing | Measure actual audio file duration | Resolves chicken-and-egg with animation timing |
| Vector store | ChromaDB (local) | No server needed, integrates with LangChain |
| HITL routing | LLM-based scene/agent classification | Natural language revision instructions |
| Example validation | Actual Manim render before indexing | Prevents RAG from amplifying hallucinations |

## Troubleshooting

**Docker not found**: Ensure Docker Desktop is running and `docker` CLI is in your PATH.

**Manim render failures**: Check `output/state.json` for error details. The self-correction
loop handles most errors automatically (up to 5 attempts per scene).

**RAG index empty**: Run `make validate-examples` first, then `make build-rag`.

**TTS errors**: Verify your `OPENAI_API_KEY` is valid and has TTS quota.
